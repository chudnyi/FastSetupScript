# FastSetupScript — сверхбыстрая первоначальная настройка сервера!

Скрипт для настройки базовых параметров сервера в один клик: создание пользователя, настройка sudo и sudo без пароля, включение входа только по публичному ключу для повышения безопасности, смена зеркал репозиториев на более быстрые и т. п.

## Быстрый старт

Запуск отдельной командой:

```bash
# Показать справку и список доступных команд
bash fss.sh

# Полная настройка (обновление + пользователь + SSH + hostname + DNS)
bash fss.sh --for-real all

# Только создать пользователя
bash fss.sh --for-real user deploy

# Только установить SSH-ключ
bash fss.sh --for-real ssh "ssh-ed25519 AAAA... user@host"

# Только изменить имя хоста
bash fss.sh --for-real --hostname node1.example.com hostname
```

Удалённый запуск (без клонирования):

```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/Bryant-Xue/FastSetupScript/main/fss.sh)" -- --for-real all
```

## Команды

| Команда           | Описание                                                       |
| ----------------- | -------------------------------------------------------------- |
| `update`          | Обновить систему (apt update && apt upgrade)                   |
| `user [ИМЯ]`      | Создать пользователя и настроить sudo NOPASSWD                |
| `ssh [КЛЮЧ]`      | Установить публичный SSH-ключ и отключить вход по паролю       |
| `hostname [ИМЯ]`  | Изменить имя хоста                                             |
| `dns`             | Синхронизировать DNS через Cloudflare (нужен `--with-dns`)     |
| `all`             | Выполнить все шаги по порядку (полная настройка)               |
| `help`            | Показать справку                                               |

## Глобальные флаги (ставятся ПЕРЕД командой)

- `--for-real` — скоростной режим без интерактивных запросов
- `--without-gum` — не использовать gum
- `--with-dns` — включить шаг синхронизации DNS
- `--hostname ИМЯ` — задать имя хоста по умолчанию
- `-h`, `--help` — показать справку

## Переменные окружения

- `SET_USER` — имя пользователя (по умолчанию: `admin`)
- `SET_HASH` — хеш пароля (SHA-512 crypt)
- `SET_KEY` — публичный SSH-ключ
- `SET_HOSTNAME` — имя хоста
- `SET_CF_ZONE_ID` — ID зоны Cloudflare
- `SET_CF_ENC` — зашифрованный API-токен Cloudflare
- `SET_TOKEN_PASS` — пароль для расшифровки токена CF

## Примеры

```bash
# Полная интерактивная настройка
bash fss.sh all

# Полная настройка в автоматическом режиме (все параметры из env)
SET_USER=deploy SET_HASH='$6$...' SET_KEY='ssh-ed25519 ...' bash fss.sh --for-real all

# Создать только пользователя
bash fss.sh user alice

# Установить только SSH-ключ и сменить имя хоста
bash fss.sh --hostname node1.example.com ssh "ssh-ed25519 AAAA... me@home"
```

### Внимание: после отключения входа по паролю убедитесь, что вход по ключу работает корректно, иначе вы потеряете доступ к серверу!
### Скрипт пока не прошёл широкого тестирования — используйте с осторожностью!
